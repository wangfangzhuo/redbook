# 行程路线规划

用户会给你一个打卡地点列表，**第一个地点是出发地/酒店**。

## 你的任务

### 第一步：并行搜索每个地点信息

对列表中每个地点（除出发地外）并行 WebSearch，搜索内容：
- 营业时间 / 开放时间
- 建议游览时长
- 门票价格（如有）
- 网友评价亮点和注意事项（1-2条）
- 推荐系数（根据搜索到的评价综合判断，1-5星）

搜索关键词格式：`[地点名] 营业时间 游览攻略` 或英文名 + `opening hours review`

### 第二步：计算步行距离

根据地理知识或搜索，估算相邻地点之间的步行距离（米）和步行时间（分钟）。
如某段距离超过 2 公里，标注"建议打车"并给出大概车程。

### 第三步：路线排序优化

排序原则（按优先级）：
1. **营业时间约束优先**：早关门的必须排前面，不能错过
2. **地理相邻**：减少来回折返，步行总距离尽量短
3. **体力节奏**：重度景点（寺庙、博物馆）放上午，购物/市场放下午

出发时间默认 08:00，除非用户指定。

### 第四步：输出行程

按以下格式输出，**与小红书帖子格式一致**：

```
🕗 [时间]  [推荐系数⭐] [地点类型标签]
📍 [地点名]
[营业时间] · [门票] · [建议游览X分钟]
[1句亮点] + [1条注意事项（如有）]

↓ 步行约Xkm · 约X分钟（或 打车约X分钟）
```

末尾附上：
- 推荐系数总览表（地点 / ⭐评分 / 建议是否保留）
- 全天步行总距离和总时长
- 如有营业时间冲突或时间紧张，单独提醒
- **固定收尾提示**（每次必须输出）：

```
⚠️ 出发前请确认
以上营业时间来自网络搜索，实际可能因节假日、季节或临时调整而变化。
建议出发前 Google Maps 搜索每个地点确认当日状态，尤其是：
- 周末/节假日营业时间与平日可能不同
- 按摩/体验类场所建议提前电话确认是否营业
- 夜市类场所受天气、学校假期影响较大
```

### 注意事项

- 推荐系数基于搜索到的网友评价综合判断，标注依据（如"Google 4.7分"、"小红书高频推荐"）
- 营业时间若搜索结果存在多个不同版本，**全部列出**并标注"⚠️ 数据有分歧，建议出发前确认"
- 保持口语化风格，和用户之前的帖子语气一致
- 不要自作主张删减用户给的地点，全部列出，让用户自己决定取舍
